# 037 - 마법학교 경영 (Magic Academy Tycoon)

## 컨셉
마법학교의 교장이 되어 학교를 경영하는 타이쿤 게임. 학생 모집, 교과과정 설계, 교사 고용, 건물 건설, 예산 관리 등 복합적인 경영 시스템을 통해 최고의 마법학교를 만든다. 핵심 재미는 다양한 성격과 재능을 가진 학생들이 성장하는 과정을 지켜보고, 학교의 명성을 높이며 라이벌 학교와 경쟁하는 데 있다. 학생 개개인의 스토리와 시스템 간의 창발적 상호작용이 30분 이상의 리플레이 가치를 제공한다.

핵심 루프: 학기 시작 → 학생 모집 → 시간표 배정 → 수업 진행(자동) → 이벤트 대응 → 시험/평가 → 학기 종료 → 졸업/재등록 → 예산 정산 → 반복

## 시스템 설계

### 시스템 1: 학생 시스템
학생은 게임의 핵심 자원이자 콘텐츠. 각 학생은 절차적으로 생성된 고유한 프로필을 가진다.

**학생 데이터 구조:**
```javascript
student = {
  id: 'STU_001',
  name: '김유리',              // 절차적 생성 (성 50개 × 이름 80개)
  portrait: { hair: 0-7, face: 0-5, robe: 0-4 }, // CSS 기반 초상화
  year: 1,                      // 1~4학년 (4년제)
  age: 15,                      // 15~18
  
  // 마법 적성 (각 0~100, 합산 평균 50)
  talents: {
    fire: 65,      // 화염
    ice: 30,       // 빙결
    lightning: 45, // 번개
    healing: 80,   // 치유
    summoning: 20, // 소환
    enchantment: 55, // 부여
    illusion: 40,  // 환영
    divination: 70, // 점술
    alchemy: 35,   // 연금술
    necromancy: 10, // 강령술
    nature: 50,    // 자연
    arcane: 60     // 신비
  },
  
  // 능력치 (0~100)
  stats: {
    intelligence: 72,  // 학습 속도 보정
    diligence: 55,     // 자습 효과, 결석률
    creativity: 68,    // 연구/실험 성과
    social: 45,        // 친구 관계 형성률
    courage: 60        // 대회/이벤트 참여율
  },
  
  // 성격 (5가지 중 1~2개)
  personality: ['perfectionist', 'rebel'],
  // perfectionist: 성적↑ 스트레스↑
  // rebel: 만족도 변동 큼, 창의성↑
  // bookworm: 도서관 보너스, 사회성↓
  // social_butterfly: 친구 多, 집중력↓
  // ambitious: 동기부여↑, 질투심↑
  // lazy: 자습 안함, 재능 높으면 천재형
  // kind: 치유 적성↑, 다른 학생 만족도↑
  // prankster: 이벤트 유발, 매력↑
  
  // 상태
  satisfaction: 70,   // 만족도 (0~100, 30 미만이면 퇴학 위험)
  stress: 20,         // 스트레스 (0~100, 80 이상이면 사고 확률↑)
  grades: {},         // 과목별 성적 { fire: 'B+', healing: 'A' }
  gpa: 3.2,           // 평점 (0.0~4.0)
  
  // 관계
  friends: ['STU_005', 'STU_012'],
  rivals: ['STU_008'],
  clubId: 'CLUB_02',  // 소속 동아리
  
  // 진행
  tuitionPaid: true,
  scholarship: false,  // 장학금 수혜 여부
  graduated: false,
  expelledRisk: false
}
```

**학생 생성 알고리즘:**
1. 재능: 12개 마법 분야 각각에 `Math.floor(Math.random() * 60) + 20` (20~80)
2. 주특기 1~2개를 선택하여 +20 보너스 (최대 100)
3. 약점 1~2개를 선택하여 -15 패널티 (최소 5)
4. 능력치: 정규분포 `Math.floor(gaussianRandom() * 20 + 50)` (30~70)
5. 성격: 8가지 중 랜덤 1~2개, 상충되는 조합 불가 (perfectionist + lazy 등)
6. 만족도 초기값: 60 + Math.random() * 20

**학생 성적 계산 (매 수업 후):**
```
성적점수 = (재능 × 0.4 + 교사능력 × 0.3 + 시설레벨 × 0.2 + 근면성 × 0.1) × (1 - 스트레스/200)
성적등급: S(95+), A+(90), A(85), B+(80), B(75), C+(70), C(65), D(60), F(60미만)
GPA = 모든 과목 점수 평균 / 25 (0.0~4.0 스케일)
```

### 시스템 2: 교과과정 & 수업 시스템
학교의 시간표를 설계하고 과목을 개설한다.

**마법 분야 12개:**

| 분야 | 아이콘 | 설명 | 선수과목 | 기본난이도 |
|------|--------|------|---------|-----------|
| 화염술 | 🔥 | 공격 마법의 기초 | 없음 | 3 |
| 빙결술 | ❄️ | 방어/제어 마법 | 없음 | 3 |
| 번개술 | ⚡ | 고급 공격 마법 | 화염술 기초 | 5 |
| 치유술 | 💚 | 회복 마법 | 없음 | 2 |
| 소환술 | 🌀 | 마물/정령 소환 | 신비학 기초 | 7 |
| 부여술 | ✨ | 아이템 강화 | 연금술 기초 | 6 |
| 환영술 | 👁️ | 감각 조작 | 없음 | 4 |
| 점술 | 🔮 | 미래/과거 감지 | 없음 | 4 |
| 연금술 | ⚗️ | 물질 변환/약품 | 없음 | 5 |
| 강령술 | 💀 | 어둠 마법 (위험) | 치유술+소환술 | 9 |
| 자연술 | 🌿 | 자연 조작 | 없음 | 3 |
| 신비학 | 📚 | 이론/기초 | 없음 | 1 |

**각 분야의 수업 단계 (4단계):**
- 기초 (1학년): 재능 요구 없음, 기본 이론
- 중급 (2학년): 재능 30 이상, 기초 C+ 이상 필요
- 고급 (3학년): 재능 50 이상, 중급 B 이상 필요
- 마스터 (4학년): 재능 70 이상, 고급 A 이상 필요

**시간표 설계:**
- 1학기 = 게임 내 5분 (60틱, 1틱=5초)
- 1주일 = 6틱 (월~토, 일요일 휴식)
- 1학기 = 10주
- 각 요일에 3교시 배정 가능
- 교시별 과목+교사+교실 배정

**수업 효과 공식:**
```
학습량 = 기본학습(10) × 교사보정(0.5~2.0) × 시설보정(0.7~1.5) × 학생재능보정(talent/50) × 집중보정
집중보정 = (100 - stress) / 100 × (diligence / 80)
```

### 시스템 3: 교사 시스템
교사를 고용하고 관리한다.

**교사 데이터 구조:**
```javascript
teacher = {
  id: 'TCH_001',
  name: '아르카두스 교수',
  specialty: 'fire',         // 전문 분야
  subSpecialty: 'lightning',  // 부전공
  
  skill: 85,          // 교수 능력 (1~100)
  research: 70,       // 연구 능력
  popularity: 60,     // 학생 인기도 (만족도 영향)
  
  salary: 500,        // 월급 (골드)
  contractYears: 3,   // 계약 기간
  yearsWorked: 1,     // 근무 연차
  
  // 특성
  traits: ['strict', 'brilliant'],
  // strict: 성적↑ 만족도↓
  // lenient: 만족도↑ 성적↓
  // brilliant: 연구↑ 급여 요구↑
  // eccentric: 수업 변동 큼, 가끔 대성공
  // dedicated: 초과 근무 가능
  // burned_out: 효과 -30% (연차 5+ 확률적)
  
  mood: 80,           // 사기 (0~100)
  maxClasses: 4       // 주당 최대 수업 수
}
```

**교사 등급 & 급여 테이블:**

| 등급 | 스킬 범위 | 월급 범위 | 출현 확률 |
|------|----------|----------|----------|
| 견습 | 20~40 | 100~200 | 40% |
| 정규 | 41~60 | 200~400 | 30% |
| 숙련 | 61~80 | 400~700 | 20% |
| 대가 | 81~95 | 700~1200 | 8% |
| 전설 | 96~100 | 1500~2500 | 2% |

**교사 고용 시스템:**
- 매 학기 시작 전 3~6명의 지원자 풀 생성
- 학교 명성이 높을수록 고급 교사 지원 확률↑
  - 명성 80+: 대가/전설 출현률 2배
- 계약 만료 시 재계약 협상 (급여 10~30% 인상 요구)
- 해고 시 명성 -5, 잔여 급여 50% 위약금

### 시스템 4: 건물 & 시설 시스템

**건물 목록:**

- 그리드 8×8 = 64칸, 초기 사용 가능 16칸 (4×4 중앙)

### 시스템 6: 학생 사회 역학 시스템
학생들 간의 관계가 자동으로 형성되고 영향을 미친다.

**관계 시스템:**
- 같은 수업을 듣는 학생끼리 매 수업마다 친밀도 +1~3
- 같은 동아리 소속: 매주 +2
- 성격 궁합: 유사 성격 +보너스, 대립 성격은 라이벌 형성
- 친밀도 50 이상: 친구 관계 (서로의 만족도 +5)
- 친밀도 -20 이하: 라이벌 (스트레스 +5, 경쟁으로 성적↑ 가능)

**동아리 (최대 5개 운영):**

| 동아리 | 활동 | 보너스 |
|--------|------|--------|
| 마법결투부 | 전투 마법 훈련 | 화염/빙결/번개 +5 |
| 치유봉사단 | 지역 봉사 | 치유 +5, 명성 +2 |
| 연금술연구회 | 포션 제조 | 연금술 +5, 수입 +소량 |
| 신비탐구회 | 고대문헌 연구 | 신비학/점술 +5 |
| 소환수호대 | 소환 연습 | 소환 +5 |
| 마법체육부 | 마법 스포츠 | 스트레스 -10 |
| 마법음악부 | 마법 연주 | 만족도 +5 |
| 자연보호단 | 생태계 관리 | 자연술 +5 |

### 시스템 8: 라이벌 학교 시스템
3개의 AI 라이벌 학교와 경쟁한다.

**라이벌 학교:**

| 학교 | 특징 | 강점 | 약점 |
|------|------|------|------|
| 불꽃탑 아카데미 | 공격 마법 특화 | 화염/번개 | 치유/자연 |
| 달빛의 학원 | 균형 잡힌 명문 | 높은 명성 | 혁신 부족 |
| 심연의 서원 | 금지 마법 연구 | 강령/소환 | 낮은 명성, 사고↑ |

**경쟁 시스템:**
- 연간 마법 대회: 학생 실력으로 승패 결정
- 명성 순위: 4개 학교 중 순위 표시
- 학생 유치 경쟁: 순위 높은 학교가 우수 학생 획득 우선권
- 라이벌 학교 AI: 매 학기 자동으로 발전 (명성 +3~8/학기)

**대회 전투 공식:**
```
학생전투력 = (해당분야재능 × 0.5 + 성적점수 × 0.3 + 용기 × 0.2) × (1 + 랜덤(-0.1, 0.1))
팀전투력 = 참가 학생 전투력 합 × 팀워크보정(친구관계 수 × 0.05)
```

### 시스템 9: 연구 시스템
새로운 마법과 기술을 연구한다.

**연구 트리:**
```
기초 마법 이론 → 복합 원소 조합 → 고급 변환술 → 궁극 마법
              → 마법 효율화 → 자동 마법진 → 마법 공학
              → 생체 마법 → 영혼 결합 → 금지된 지식
```

### 시스템 10: 인증 & 명성 시스템

**명성 점수 (0~100):**
```
명성 = 기본(10) + 건물보정 + 교사보정 + 졸업생보정 + 대회보정 + 연구보정 - 사고패널티
건물보정: 마법탑 ×10 + 총건물레벨 × 1
교사보정: 대가이상 교사 수 × 5
졸업생보정: 누적 졸업생 × 0.5 (최대 20)
대회보정: 우승 횟수 × 3
연구보정: 완료 연구 수 × 2
사고패널티: 최근 3학기 사고 수 × 3
```

**인증 등급:**

| 등급 | 요구 명성 | 요구 졸업률 | 보너스 |
|------|----------|-----------|--------|
| 미인증 | - | - | 없음 |
| 기본 인증 | 20 | 50% | 보조금 500 |
| 우수 인증 | 50 | 70% | 보조금 1000, 학비 +20% |
| 최우수 인증 | 80 | 85% | 보조금 2000, 학비 +50%, 천재 학생↑ |
| 왕립 인가 | 95 | 95% | 보조금 5000, 전설 교사 지원↑ |

- **차트:** Canvas 기반 레이더차트, 바차트

### 사운드 (Web Audio API)
- **BGM:** 미니멀 앰비언트 루프 (오실레이터 기반 판타지 느낌)
  - 삼각파 + 사인파 코드 진행: Am → F → C → G (120BPM, 천천히)
- **효과음:**
  - 건물 건설: 낮은 주파수 타격음 (100Hz → 50Hz 페이드)
  - 학기 시작: 종소리 (880Hz 사인파 × 2 반복)
  - 이벤트 발생: 경고음 (440Hz → 880Hz 상승)
  - 성적 발표: 파열음 (화이트노이즈 0.1초)
  - 졸업: 팡파레 (도-미-솔 아르페지오)
  - 골드 획득: 동전 소리 (2000Hz 짧은 삼각파)
  - 사고: 폭발음 (화이트노이즈 + 저주파 럼블)
  - UI 클릭: 짧은 틱 (1000Hz 사각파 0.02초)

6. "학기를 시작합니다" — 게임 흐름 설명
7. "학생 상태를 확인하세요" — 만족도/스트레스 관리
8. "이벤트에 대응하세요" — 첫 이벤트 대응
9. "동아리를 만들어보세요" — 동아리 시스템
10. "연구를 시작합시다" — 연구 시스템
11. "대회에 참가합시다" — 대회 시스템
12. "학기말 보고서" — 보고서 읽기
13. "자유 플레이!" — 튜토리얼 완료

각 단계는 해당 UI 요소 하이라이트 + 화살표 표시 + 설명 팝업으로 구성.

## 기술 도전 및 해결

### 도전 1: 대규모 학생 시뮬레이션 성능
- **문제:** 40명 학생 × 관계 × 매 틱 업데이트 시 성능
- **해결:** 틱 간격 5초, 관계 업데이트는 주 단위, dirty flag로 변경 시만 UI 갱신

### 도전 2: 시간표 충돌 검증
- **문제:** 교사/교실/학생의 다중 제약 조건 검증
- **해결:** 슬롯별 할당 맵 유지, O(1) 충돌 확인

### 도전 3: 절차적 학생 생성의 균형
- **문제:** 완전 랜덤 시 비현실적 능력치 조합
- **해결:** 재능 합산 제한 (평균 50 ± 10), 성격-능력치 연동

### 도전 4: 단일 파일 크기 관리
- **문제:** 1000줄+ 게임 로직을 단일 파일에 유지
- **해결:** 모듈 패턴 (IIFE), 데이터 테이블 상수 분리, CSS는 `<style>` 태그 내 최소화

### 도전 5: 저장 데이터 마이그레이션
- **문제:** 업데이트 시 기존 세이브 호환성
- **해결:** version 필드로 마이그레이션 함수 체인 적용
