# 028 - 프로그래밍 퍼즐

## 컨셉
비주얼 프로그래밍으로 로봇에게 명령을 내려 목표를 달성하는 퍼즐 게임. 조건문, 반복문, 함수 등의 프로그래밍 개념을 블록을 드래그&드롭하여 배치하고, 실행 버튼을 눌러 로봇이 명령대로 움직이는 것을 관찰한다. Lightbot + Scratch의 교육적 재미를 게임화. 총 25개 레벨로 프로그래밍의 핵심 개념을 자연스럽게 학습.

핵심 재미: "내가 짠 프로그램이 작동하는" 성취감 + 최소 블록으로 해결하는 최적화 도전 + 점진적 난이도 상승으로 자연스러운 프로그래밍 학습.

## 핵심 시스템

  - 최대 블록 수 제한 (레벨별)

### 시스템 2: 그리드 월드 & 로봇
- **그리드**: 레벨별 크기 (4x4 ~ 10x10)
- **타일 종류**:
  | 타일 | 표현 | 설명 |
  |------|------|------|
  | 바닥 | 밝은 회색 | 이동 가능 |
  | 벽 | 진한 회색 | 이동 불가 |
  | 구멍 | 검정 | 이동 불가, 아이템으로 메움 가능 |
  | 목표 지점 | 노란 별 | 로봇이 도달해야 할 위치 |
  | 아이템 칸 | 보석 아이콘 | 수집할 아이템 |
  | 색칠 칸 | 회색 → 색 | 칠하기로 색 변경 |
  | 텔레포터 | 보라색 | 짝 텔레포터로 순간이동 |
  | 얼음 | 하늘색 | 밟으면 미끄러짐 (같은 방향 계속 이동) |

- **로봇 상태**:
  ```
  robot = {
    x, y,                    // 위치
    direction: 0,            // 0=북, 1=동, 2=남, 3=서
    bag: [],                 // 수집한 아이템
    bagCapacity: 3,          // 가방 용량
    executing: false,        // 실행 중 여부
    currentBlock: 0,         // 현재 실행 중인 블록 인덱스
    stepCount: 0,            // 총 실행 스텝 수
    maxSteps: 200            // 무한루프 방지 제한
  }
  ```

### 시스템 3: 실행 엔진 & 시각화
- **실행 모드**:
  - ▶ 실행: 0.3초 간격으로 한 블록씩 실행. 현재 실행 중인 블록 하이라이트.
  - ⏩ 빠른 실행: 0.05초 간격.
  - ⏭ 스텝 실행: 버튼 누를 때마다 한 블록 실행.
  - ⏹ 정지: 실행 중단, 초기 상태로 복원.

- **실행 엔진 (인터프리터)**:
  ```
  executeProgram(blocks):
    callStack = [{ blocks, index: 0 }]
    while callStack is not empty:
      frame = callStack.top()
      if frame.index >= frame.blocks.length:
        callStack.pop()
        continue
      block = frame.blocks[frame.index]
      frame.index++
      
      switch block.type:
        'forward': moveRobot(direction)
        'turnLeft': robot.direction = (robot.direction + 3) % 4
        'turnRight': robot.direction = (robot.direction + 1) % 4
        'pickup': pickupItem()
        'drop': dropItem()
        'paint': paintTile(block.color)
        'repeat': 
          for i in range(block.count):
            callStack.push({ blocks: block.children, index: 0 })
        'if':
          if evaluateCondition(block.condition):
            callStack.push({ blocks: block.children, index: 0 })
        'ifelse':
          if evaluateCondition(block.condition):
            callStack.push({ blocks: block.trueChildren, index: 0 })
          else:
            callStack.push({ blocks: block.falseChildren, index: 0 })
        'funcDef': // 저장만
        'funcCall': 
          callStack.push({ blocks: functions[block.funcId], index: 0 })
      
      yield // 한 스텝마다 대기 (시각화)
      if stepCount > maxSteps: abort("무한 루프!")
  ```

- **실행 시각화**:
  - 로봇 이동: 부드러운 lerp 애니메이션 (200ms)
  - 로봇 회전: 90도 회전 애니메이션 (150ms)
  - 아이템 줍기: 아이템이 로봇 위로 떠올라 사라짐
  - 칠하기: 타일 색상이 물결처럼 퍼짐
  - 현재 블록: 노란 테두리 하이라이트 + 약간의 스케일업
  - 실패 시: 로봇 흔들림 + 빨간 X 표시
  - 성공 시: 별 파티클 + 축하 애니메이션

### 시스템 4: 별점 & 도전 시스템
- **레벨 클리어 조건**: 레벨별로 다름 (목표 도달, 아이템 전부 수집, 모든 칸 칠하기 등)
- **별점**:
  - 별 1: 클리어
  - 별 2: 블록 N개 이하로 클리어
  - 별 3: 스텝 M 이하로 클리어
- **도전 과제** (전체):
  - 모든 레벨 별 3개 달성
  - 함수만으로 레벨 20+ 클리어
  - 전체 블록 사용 수 100 이하로 모든 레벨 클리어

## 게임 규칙
- 블록을 배치한 후 실행 버튼으로 로봇이 프로그램을 수행
- 로봇은 블록 순서대로 한 동작씩 수행
- 벽이나 맵 밖으로 이동 시 "충돌!" 실패
- 구멍에 빠지면 실패 (아이템을 놓아 메울 수 있음)
- 최대 스텝 200 초과 시 "무한 루프!" 실패
- 가방이 가득 찬 상태에서 줍기 시 무시
- 텔레포터는 짝이 있어야 작동 (없으면 일반 바닥처럼 작동)
- 얼음 위에서는 벽이나 일반 바닥에 닿을 때까지 미끄러짐
- 실행 중에는 블록 편집 불가
- 레벨 선택 화면에서 해금된 레벨만 플레이 가능
- 이전 레벨을 재도전하여 더 좋은 별점을 받을 수 있음
- 블록 제한이 있는 레벨에서는 팔레트에 남은 수 표시

## AI 설계

### 힌트 시스템 (규칙 기반)
플레이어가 5회 이상 실패하면 단계적 힌트 제공:

```
힌트 레벨 1 (5회 실패):
  - "이 레벨의 핵심은 [반복/조건/함수] 개념입니다"

힌트 레벨 2 (8회 실패):
  - "로봇이 [방향]으로 [N]칸 이동해야 합니다"
  - 첫 번째 블록을 팔레트에서 깜빡이게 표시

힌트 레벨 3 (12회 실패):
  - "정답은 아니지만 참고하세요" → 부분 솔루션 보여주기
  - 첫 3개 블록을 자동 배치
```

### 솔루션 검증기 AI
레벨 설계 시 각 레벨의 최적 솔루션을 사전 정의. 플레이어 솔루션과 비교:
```
- 최적 블록 수 계산 (BFS로 최소 이동 경로 기반)
- 별 2 기준 = 최적 * 1.5
- 별 3 기준 = 최적 스텝 * 1.3
```

### 절차적 보너스 레벨 생성
레벨 25 클리어 후 무한 보너스 레벨:
```
1. 그리드 크기 랜덤 (5x5 ~ 8x8)
2. 시작점 = (0, 0), 목표점 = 대각선 반대편
3. 벽을 랜덤 배치 (전체 칸의 20~30%)
4. 경로 존재 여부 BFS로 검증
5. 아이템/색칠/텔레포터 중 1~2개 랜덤 추가
6. 난이도 = 최적 경로 길이 * 장애물 수
```

## 기술 구현 계획

### 도전 과제 및 해결
1. **블록 드래그&드롭**: HTML5 Drag API 대신 커스텀 구현 (모바일 호환). mousedown/touchstart → mousemove/touchmove → mouseup/touchend. 블록 위치를 절대 좌표로 추적.
2. **중첩 블록**: 블록 데이터를 트리 구조로 관리. 반복/조건 블록의 children 배열에 자식 블록. 렌더링 시 재귀적으로 그리기. 드롭 존 계산을 재귀적으로.
3. **인터프리터**: 제너레이터 함수(function*)로 구현하여 한 스텝씩 yield. 애니메이션 완료 후 다음 스텝 실행.
4. **애니메이션 큐**: 로봇 이동 애니메이션을 Promise로 래핑. await하여 순차 실행.
5. **터치 호환**: 모든 마우스 이벤트에 대응하는 터치 이벤트 핸들러. pointer events 활용.
