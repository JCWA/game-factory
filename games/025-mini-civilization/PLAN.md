# 025 - 미니 문명 시뮬레이터

## 컨셉
턴제 전략 게임. 헥스 타일맵 위에서 도시를 건설하고, 자원을 관리하며, 기술을 연구하고, 적 문명 AI와 외교 또는 전쟁을 벌인다. 문명 시리즈의 핵심 재미를 단일 HTML 파일에 압축한 게임. "4X" (eXplore, eXpand, eXploit, eXterminate) 장르의 정수를 담되, 30분 내로 한 판이 끝나도록 설계한다.

핵심 재미: 제한된 자원과 턴 속에서 기술 트리/군사력/외교의 균형을 잡는 전략적 의사결정. 매 판 다른 맵과 AI 성향으로 리플레이성 확보.

## 핵심 시스템

### 시스템 1: 헥스 타일맵 & 절차적 맵 생성
- **맵 크기**: 20x15 헥스 타일 (총 300타일)
- **지형 타입 7종**:
  | 지형 | 식량 | 생산력 | 금 | 이동비용 | 색상 |
  |------|------|--------|---|---------|------|
  | 평원 | 2 | 1 | 0 | 1 | #90BE6D |
  | 초원 | 3 | 0 | 0 | 1 | #43AA8B |
  | 숲 | 1 | 2 | 0 | 2 | #2D6A4F |
  | 산 | 0 | 3 | 1 | 3 | #6C757D |
  | 사막 | 0 | 1 | 2 | 1 | #F4D35E |
  | 물 | 0 | 0 | 1 | ∞(통행불가) | #277DA1 |
  | 언덕 | 1 | 2 | 0 | 2 | #B5838D |

- **절차적 생성 알고리즘**:
  1. Perlin Noise로 높이맵 생성 (simplex noise 근사 구현)
  2. 높이 임계값으로 지형 분류: <0.25 물, <0.40 평원, <0.55 초원, <0.65 숲, <0.80 언덕, <0.90 산, ≥0.90 사막(고온지대)
  3. 각 문명 시작점은 최소 8타일 거리, 물이 아닌 곳, 식량 합 7 이상인 주변 보장
  4. 특수 자원 (말, 철, 보석) 랜덤 배치 — 맵당 각 3-5개

- **헥스 좌표**: offset coordinates 사용. 홀수 행 오른쪽으로 0.5 오프셋.
- **시야 시스템**: 탐험하지 않은 타일은 검은색. 유닛/도시 주변 2칸은 밝게, 이전에 본 타일은 안개(반투명 회색).

### 시스템 2: 도시 & 경제
- **도시 속성**:
  - 인구 (1~20), 체력 (100), 방어력
  - 매 턴 자원 산출: 주변 타일(반경 2칸)의 식량/생산력/금 합산
  - 인구 성장: 누적 식량이 `인구 * 15`에 도달하면 인구 +1
  - 생산 큐: 건물/유닛을 큐에 넣으면 생산력 누적으로 완성

- **건물 8종**:
  | 건물 | 생산비용 | 효과 |
  |------|---------|------|
  | 곡물창고 | 30 | 식량 +3/턴 |
  | 작업장 | 40 | 생산력 +3/턴 |
  | 시장 | 50 | 금 +4/턴 |
  | 병영 | 40 | 군사유닛 생산속도 +50% |
  | 성벽 | 60 | 도시 방어력 +5, 체력 +50 |
  | 도서관 | 50 | 연구력 +3/턴 |
  | 신전 | 60 | 행복도 +2 |
  | 궁전 | 80 | 모든 자원 +1/턴 (수도만) |

- **행복도 시스템**: 기본 5. 인구 1당 -1. 건물/정책으로 보정. 행복도 < 0이면 생산력 -50%.

### 시스템 3: 기술 연구 트리
- **3개 시대** (고대 → 중세 → 근대), 총 18개 기술
- **연구력**: 도시의 도서관 + 기본값(인구/2)으로 매 턴 누적

```
[고대 시대]
농업(20) → 관개(35) → 축산(50)
채광(20) → 제련(35) → 대장장이(50)
전투술(20) → 궁술(35) → 전략(50)

[중세 시대] (고대 3개 이상 필요)
기사도(80) → 화약(120)
항해술(80) → 탐험(120)
건축학(80) → 도시계획(120)

[근대 시대] (중세 3개 이상 필요)
산업혁명(200) → 철도(300)
민주주의(200) → 자유무역(300)
```

- 각 기술은 새 건물, 유닛, 또는 보너스를 해금

### 시스템 4: 군사 & 전투
- **유닛 5종**:
  | 유닛 | 공격 | 방어 | 이동 | 생산비용 | 필요기술 |
  |------|------|------|------|---------|---------|
  | 전사 | 5 | 3 | 2 | 20 | 없음 |
  | 궁수 | 7 | 2 | 2 | 30 | 궁술 |
  | 기사 | 10 | 5 | 3 | 50 | 기사도 |
  | 공성추 | 15 | 2 | 1 | 60 | 화약 |
  | 머스킷병 | 12 | 8 | 2 | 70 | 산업혁명 |

- **전투 공식**: 
  ```
  공격자 피해 = (공격자.공격 * random(0.8~1.2)) - (방어자.방어 * 0.5)
  방어자 반격 = (방어자.공격 * 0.5 * random(0.8~1.2)) - (공격자.방어 * 0.3)
  지형 보너스: 숲/언덕 +2 방어, 산 +4 방어, 도시 +도시방어력
  ```
- 유닛 체력 20. 체력 0 이하면 파괴.

### 시스템 5: 외교 AI
- **AI 문명 2개** (3인 플레이)
- **AI 성향** (게임 시작 시 랜덤):
  - 공격적 (군사 우선, 조기 전쟁)
  - 확장적 (도시 다수 건설, 영토 중시)
  - 학자적 (기술 우선, 방어적)

- **외교 상태**: 중립 → 우호(교역 가능) → 동맹 / 중립 → 적대 → 전쟁
- **외교 액션**: 교역 제안 (금 ↔ 자원), 평화 조약, 선전포고
- **호감도 계산**:
  ```
  기본 호감도 = 50
  군사력 차이 > 2배 → -20 (위협)
  국경 인접 → -10
  교역 중 → +15
  전쟁 이력 → -30
  같은 적과 전쟁 중 → +20
  ```

## 게임 규칙
- 게임 시작 시 플레이어와 AI 2개 문명이 맵에 배치됨
- 각 문명은 개척자 1명 + 전사 1명으로 시작
- 개척자를 원하는 위치에서 도시로 전환 가능
- 턴제: 유닛 이동 → 도시 관리 → 연구 선택 → 턴 종료
- 식량이 부족하면 인구가 감소 (기아)
- 금이 음수이면 유닛이 매 턴 체력 -5 (유지비 부족)
- 유닛은 한 턴에 이동력만큼 이동 가능
- 도시를 공격해 체력을 0으로 만들면 점령 가능
- 모든 적 문명의 수도를 점령하면 군사 승리
- 근대 시대 기술 4개 이상 연구하면 기술 승리
- 100턴 경과 시 점수(도시 수 + 인구 + 기술 수 + 군사력)로 판정
- 맵 가장자리는 통행 불가 (경계)
- 같은 타일에 아군 유닛 2개 이상 불가 (스택 금지)
- 유닛은 매 턴 자동 회복 +2 HP (도시 위에선 +5)

## AI 설계

### AI 의사결정 (유틸리티 AI)
매 턴 AI는 다음 행동 후보에 점수를 매기고 가장 높은 것을 실행:

```
행동 후보 목록:
1. 도시 건설 (개척자 생산 → 이동 → 건설)
   점수 = (도시 수 < 3) ? 80 : 20 - 도시수 * 10

2. 군사 유닛 생산
   점수 = max(0, 50 - 아군유닛수 * 10 + 위협레벨 * 20)

3. 건물 건설
   점수 = 40 + (식량부족 ? 20 : 0) + (금부족 ? 15 : 0)

4. 기술 연구
   점수 = 30 + (학자성향 ? 25 : 0)

5. 전쟁 선포
   점수 = (군사력 > 적군사력 * 1.5 && 공격성향) ? 70 : 10

6. 교역 제안
   점수 = (호감도 > 40 && 금부족) ? 50 : 15
```

### AI 유닛 이동 (A* 경로탐색)
- 공격 대상 선정: 가장 가깝고 약한 적 유닛/도시
- 방어 모드: 자기 도시 주변 2칸 이내 순찰
- 탐험 모드: 미탐험 타일 방향으로 이동

### AI 턴 처리 순서
1. 위협 평가 (적 유닛이 도시 5칸 이내?)
2. 자원 상태 확인
3. 유틸리티 점수 계산
4. 최고 점수 행동 실행
5. 유닛 명령 (전투/이동/순찰)
6. 외교 결정

## 레벨/콘텐츠 설계

### 도시 이름 풀 (플레이어)
서울, 부산, 인천, 대전, 광주, 대구, 울산, 수원

### AI 문명 이름
- 문명 1: 용의 제국 (도시: 용성, 화산, 비룡, ...)
- 문명 2: 바다의 왕국 (도시: 파도, 해류, 산호, ...)

### 특수 자원 효과
| 자원 | 출현지형 | 효과 |
|------|---------|------|
| 말 | 초원 | 기사 생산 가능, 이동력 +1 |
| 철 | 산/언덕 | 유닛 공격력 +2 |
| 보석 | 사막 | 금 +3/턴 |

### 승리 조건 상세
1. **군사 승리**: 모든 적 수도 점령
2. **기술 승리**: 근대 시대 기술 4개 달성 → "우주 프로그램" 프로젝트 완성 (200 생산력)
3. **점수 승리**: 100턴 종료 시 점수 계산
   - 도시 1개당 20점
   - 인구 1당 5점
   - 기술 1개당 10점
   - 유닛 1개당 3점
   - 금 10당 1점

## 기술 구현 계획

### 도전 과제 및 해결
1. **헥스 타일 렌더링**: 정육각형 path 그리기. pointy-top 방식. 클릭 좌표 → 헥스 좌표 변환은 pixel_to_hex 함수 (큐브 좌표 활용 후 offset 변환).
2. **A* 경로탐색**: 헥스 그리드용 A* 구현. 이웃 타일 6개, 이동비용 = 지형.이동비용. 물 타일은 통행불가.
3. **Perlin Noise**: 2D gradient noise 직접 구현 (외부 라이브러리 없이). 격자점에 랜덤 기울기 벡터 배치, 보간.
4. **AI 턴 처리**: requestAnimationFrame으로 AI 턴을 비동기 처리 (UI 블로킹 방지). AI 행동을 단계별로 나누어 각 프레임에 하나씩 실행.
5. **성능**: 300타일 + 최대 30유닛 정도이므로 Canvas 2D로 충분. dirty flag로 변경된 부분만 다시 그리기.

## 저장/불러오기

```javascript
// localStorage 키: 'miniCiv_save'
const saveData = {
  version: 1,
  timestamp: Date.now(),
  gameState: {
    turn: 15,
    civilizations: [...], // 전체 문명 데이터
    tiles: [...], // 맵 데이터 (explored 정보 포함)
    winner: null
  }
};
localStorage.setItem('miniCiv_save', JSON.stringify(saveData));
```

- 자동 저장: 매 5턴마다
- 수동 저장: 저장 버튼 클릭
- 불러오기: 메인 메뉴에서 "이어하기" 버튼
- 새 게임 시 기존 저장 데이터 확인 후 덮어쓰기 확인
