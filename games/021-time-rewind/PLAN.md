# 021 - 시간 되감기

## 컨셉
시간을 되감는 능력을 가진 캐릭터가 퍼즐을 푸는 2D 플랫포머. 과거의 자신("잔상")이 이전 행동을 반복하는 동안, 현재의 자신과 협력하여 스위치를 누르고 문을 열어 탈출한다. 핵심 메카닉은 "녹화→되감기→동시 재생"이다. 점점 복잡해지는 15개의 스테이지를 클리어한다.

## 게임 규칙
- 캐릭터는 이동, 점프, 스위치 밟기가 가능하다
- "녹화 시작" 후의 모든 행동이 기록된다
- "되감기" 버튼을 누르면 시간이 녹화 시작 지점으로 돌아간다
- 되감기 후 과거의 자신(잔상)이 녹화된 행동을 자동 반복한다
- 현재 캐릭터와 잔상이 동시에 존재하며, 각각 독립적으로 스위치를 밟을 수 있다
- 스위치를 밟으면 대응하는 문이 열린다 (밟고 있는 동안만 열림)
- 최대 3개의 잔상을 만들 수 있다 (3번 되감기)
- 출구 문에 도달하면 스테이지 클리어
- 스파이크/함정에 닿으면 사망 → 현재 녹화 취소, 마지막 되감기 지점부터 재시도
- 각 스테이지의 목표: 최소 되감기 횟수로 출구 도달

## 조작법
- PC: 화살표/WASD로 이동, 위/W로 점프, R로 녹화 시작/종료, T로 되감기, 스페이스로 스위치 상호작용
- 모바일: 왼쪽 가상 조이스틱으로 이동, 오른쪽 A버튼 점프, 상단 R/T 버튼

## 비주얼 스타일
- 네온 사이버펑크 스타일: 어두운 배경 (#0f0f1a)에 네온 라인
- 벽/바닥: 진한 회색 (#2a2a3a) + 밝은 테두리 (#4a4a6a)
- 현재 캐릭터: 밝은 시안 (#00ffff) 사각형 + 발광
- 잔상 1: 보라색 (#ff00ff) 반투명
- 잔상 2: 노란색 (#ffff00) 반투명
- 잔상 3: 초록색 (#00ff00) 반투명
- 스위치: 색상으로 대응하는 문과 매칭 (빨강 스위치→빨강 문)
- 되감기 효과: VHS 노이즈 + 화면 왜곡 애니메이션 (0.5초)
- 스파이크: 삼각형 빨간색, 위아래로 살짝 진동
- 출구: 밝은 흰색 발광 문

## 핵심 시스템

### 행동 녹화 시스템
```javascript
// 매 프레임의 입력 상태를 기록
const recording = {
  startState: { x, y },  // 녹화 시작 시 캐릭터 위치
  frames: [
    // 각 프레임의 입력 상태
    { left: false, right: true, jump: false, interact: false },
    { left: false, right: true, jump: true, interact: false },
    // ...
  ]
};

// 잔상 재생
class Ghost {
  constructor(recording) {
    this.recording = recording;
    this.frameIndex = 0;
    this.x = recording.startState.x;
    this.y = recording.startState.y;
  }
  
  update() {
    if (this.frameIndex >= this.recording.frames.length) return;
    const input = this.recording.frames[this.frameIndex];
    // input을 기반으로 물리 시뮬레이션 (실제 캐릭터와 동일한 로직)
    this.applyInput(input);
    this.frameIndex++;
  }
}
```

### 되감기 시스템
```
1. 녹화 종료 시 recording 객체를 ghosts 배열에 추가
2. 게임 상태를 녹화 시작 시점으로 복원
3. 모든 잔상을 프레임 0부터 재생 시작
4. 현재 캐릭터는 자유롭게 조작 가능
5. 최대 3개의 잔상 동시 존재
```

### 물리 시스템
```javascript
const PHYSICS = {
  gravity: 0.5,
  moveSpeed: 3,
  jumpForce: -10,
  friction: 0.85
};

function updateCharacter(char, input, level) {
  // 수평 이동
  if (input.left) char.vx = -PHYSICS.moveSpeed;
  if (input.right) char.vx = PHYSICS.moveSpeed;
  if (!input.left && !input.right) char.vx *= PHYSICS.friction;
  
  // 점프 (바닥에 있을 때만)
  if (input.jump && char.onGround) char.vy = PHYSICS.jumpForce;
  
  // 중력
  char.vy += PHYSICS.gravity;
  
  // 충돌 감지 (AABB)
  char.x += char.vx;
  resolveCollisionX(char, level);
  char.y += char.vy;
  resolveCollisionY(char, level);
}
```

### 스테이지 설계 (15개)
```
1. 튜토리얼 - 이동과 점프만으로 클리어 (되감기 불필요)
2. 스위치 기초 - 스위치 1개, 잔상 1개로 문 열기
3. 타이밍 - 잔상이 스위치를 밟는 타이밍에 맞춰 문 통과
4. 두 개의 스위치 - 잔상 2개 필요
5. 수직 이동 - 위로 올라가는 구조, 잔상과 시간차 점프
6. 스파이크 회피 - 스파이크 사이를 지나가며 스위치 조작
7. 긴 통로 - 잔상의 녹화 시간 관리가 핵심
8. 연쇄 스위치 - 스위치 3개를 순서대로 밟아야 하는 구조
9. 역방향 - 잔상이 역방향으로 가야 하는 퍼즐
10. 엘리베이터 - 스위치로 플랫폼 이동시키는 구조
11. 좁은 통로 - 정밀한 타이밍 필요
12. 미로 - 잔상 3개를 모두 활용하는 복잡한 구조
13. 함정 구간 - 스파이크가 주기적으로 나타나는 구간
14. 최종 전 - 모든 메카닉 종합
15. 최종 - 잔상 3개 + 스위치 4개 + 스파이크의 종합 퍼즐
```

## 기술 구현
- Canvas 2D로 렌더링, 타일 기반 맵 (타일 크기 32px)
- 캐릭터/잔상은 24x24 사각형, 색상과 발광으로 구분
- 충돌 감지: AABB + 타일맵 기반, 가장자리 보정
- 녹화 데이터는 프레임별 입력 상태만 저장 (위치 X → 결정론적 재현)
- 되감기 연출: CSS filter로 잠깐 화면 흐림 + 색상 반전, canvas에 스캔라인 효과
- 스위치-문 연동: 매 프레임 스위치 위에 캐릭터/잔상이 있는지 확인
- 카메라: 캐릭터 중심 추적, 맵 경계 클램핑
- 모바일 가상 조이스틱: 터치 시작 위치 기준 상대 이동량 계산
- 스테이지 전환: 페이드 아웃→인 효과
