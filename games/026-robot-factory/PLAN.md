# 026 - 로봇 팩토리

## 컨셉
자동화 공장 시뮬레이션 퍼즐 게임. 컨베이어 벨트, 가공 기계, 분류기, 합성기 등을 2D 그리드에 배치하여 원재료를 완제품으로 변환하는 생산 라인을 구축한다. Factorio + Opus Magnum의 퍼즐적 재미를 단일 HTML에 구현. 각 레벨마다 목표 제품과 수량이 주어지고, 제한된 예산과 공간 내에서 최적의 라인을 설계해야 한다.

핵심 재미: "작동하는 시스템을 설계하는" 엔지니어링의 쾌감. 최적화 욕구를 자극하는 효율 점수 시스템.

## 핵심 시스템

### 시스템 1: 그리드 & 기계 배치
- **그리드 크기**: 레벨별 다름. 최소 8x8 ~ 최대 16x12
- **배치 규칙**:
  - 기계는 1x1 또는 2x1 크기
  - 회전 가능 (0, 90, 180, 270도)
  - 입력 방향/출력 방향이 정해져 있음
  - 겹치기 불가
  - 배치 후에도 이동/삭제 가능 (시뮬레이션 정지 중에만)

- **기계 종류 12종**:

| 기계 | 크기 | 비용 | 입력 | 출력 | 기능 |
|------|------|------|------|------|------|
| 컨베이어 벨트 | 1x1 | 5 | 뒤 | 앞 | 아이템을 한 방향으로 이동 |
| 커브 벨트 | 1x1 | 8 | 뒤 | 왼/오른쪽 | 90도 방향 전환 |
| 분배기 | 1x1 | 15 | 뒤 | 양옆 | 아이템을 교대로 좌우 분배 |
| 합류기 | 1x1 | 15 | 양옆 | 앞 | 두 라인을 하나로 합침 |
| 절단기 | 1x1 | 30 | 뒤 | 앞 | 원재료를 절반으로 자름 |
| 용광로 | 2x1 | 50 | 뒤 | 앞 | 광석 → 금속 괴로 제련 (3초) |
| 프레스기 | 1x1 | 40 | 뒤 | 앞 | 금속 괴 → 판금으로 압축 (2초) |
| 조립기 | 2x1 | 60 | 양옆 | 앞 | 두 부품을 하나로 조합 (4초) |
| 도색기 | 1x1 | 35 | 뒤 | 앞 | 아이템에 색상 적용 (1초) |
| 품질검사기 | 1x1 | 25 | 뒤 | 앞/옆 | 불량품을 옆으로 분류 |
| 포장기 | 1x1 | 45 | 뒤 | 앞 | 완제품을 포장 (2초) |
| 저장소 | 1x1 | 20 | 뒤 | - | 아이템을 최대 10개 보관 |

- **원재료 공급기** (고정 위치): 레벨에 따라 1~3개. 일정 간격으로 원재료 생성.
- **출하 지점** (고정 위치): 완제품을 여기에 보내면 점수 획득.

### 시스템 2: 아이템 & 레시피 시스템
- **아이템 속성**: { type, material, color, quality, components[] }
- **기본 재료 5종**: 철광석, 구리광석, 목재, 유리, 플라스틱
- **가공 변환 테이블**:

```
철광석 → [용광로] → 철괴 → [프레스기] → 철판
구리광석 → [용광로] → 구리괴 → [프레스기] → 구리판
목재 → [절단기] → 판자
유리 → [절단기] → 유리판
플라스틱 → [프레스기] → 플라스틱 시트
```

- **레시피 (조립기)**:
```
철판 + 구리판 → 회로기판
철판 + 플라스틱 시트 → 케이싱
회로기판 + 케이싱 → 전자부품
판자 + 유리판 → 창문
철판 + 판자 → 가구 프레임
전자부품 + 케이싱 → 로봇 코어
```

- **품질 시스템**: 
  - 기본 품질 100%. 기계 과부하(입력 대기열 > 3) 시 품질 -10%
  - 품질검사기: 70% 미만 품질 아이템을 불량으로 분류
  - 출하 점수에 품질 계수 적용

### 시스템 3: 시뮬레이션 엔진
- **틱 기반 시뮬레이션**: 1틱 = 500ms (속도 조절 가능: x0.5, x1, x2, x4)
- **처리 순서** (매 틱):
  1. 원재료 공급기에서 아이템 생성
  2. 모든 기계의 가공 타이머 감소
  3. 가공 완료된 아이템 출력
  4. 컨베이어 벨트 아이템 이동 (끝에서부터 역순 처리)
  5. 출하 지점 도달 아이템 점수 계산
  6. UI 업데이트

- **병목 감지**: 기계 앞에 아이템이 쌓이면 빨간 경고 표시
- **통계 추적**: 총 생산량, 분당 처리량, 병목 위치, 유휴 기계

### 시스템 4: 레벨 & 도전 시스템
- **목표 유형**:
  - "N개의 X 제품 출하" (기본)
  - "M분 내에 N개 출하" (시간 제한)
  - "예산 B 이하로 라인 구축" (비용 제한)
  - "효율 E% 이상 달성" (최적화)

- **별점 시스템** (레벨당 3개 별):
  - 별 1: 목표 달성
  - 별 2: 시간/비용 목표 달성
  - 별 3: 완벽 효율 (유휴 기계 0, 불량률 5% 미만)

## 게임 규칙
- 시뮬레이션 시작 전에만 기계 배치/수정 가능
- 시뮬레이션 중에는 속도 조절과 일시정지만 가능
- 레벨 예산 내에서 기계를 구매해야 함
- 컨베이어 벨트는 연결되어야 아이템이 이동함
- 조립기에는 반드시 두 입력이 모두 들어와야 작동
- 출력이 막히면 기계가 정지 (백프레셔)
- 잘못된 재료가 기계에 들어가면 불량품 생성
- 불량품은 출하 시 점수에서 제외
- 레벨 클리어 시 다음 레벨 해금
- 자유 모드: 모든 기계 해금, 무제한 예산으로 자유 설계
- 각 기계는 전력을 소비 (고급 레벨에서 전력 제한 추가)
- 저장소는 아이템을 버퍼로 보관하여 흐름 조절 가능
- 분배기는 좌우 교대로 아이템 전송 (홀수 왼쪽, 짝수 오른쪽)

## AI 설계

### 자동 힌트 시스템 (규칙 기반 AI)
플레이어가 3분 이상 막혀있으면 힌트를 제공하는 AI:

```
규칙 1: 출하 지점에 도달하는 경로가 없으면
  → "컨베이어 벨트로 공급기와 출하 지점을 연결하세요"

규칙 2: 레시피에 필요한 기계가 배치되지 않았으면  
  → "이 제품에는 [기계명]이 필요합니다"

규칙 3: 조립기에 입력이 하나만 연결되어 있으면
  → "조립기에는 두 개의 입력이 필요합니다"

규칙 4: 병목이 감지되면
  → "이 기계에 아이템이 쌓이고 있습니다. 라인을 분기하세요"
```

### 절차적 레벨 생성 (고급 레벨)
레벨 15 이후로는 절차적으로 생성:
```
1. 무작위로 목표 제품 선택 (복잡도에 따라)
2. 레시피 트리에서 필요한 원재료 역추적
3. 원재료 공급기 배치 (가장자리)
4. 출하 지점 배치 (반대편 가장자리)
5. 그리드 크기 = 기본 + 복잡도 * 2
6. 예산 = 최적 솔루션 비용 * 1.5
7. 장애물 랜덤 배치 (0~3개)
```

## 조작법
- **PC**:
  - 좌측 패널에서 기계 선택 → 그리드 클릭으로 배치

  - 하단 기계 선택 바 → 탭으로 배치
  - 두 손가락 회전 또는 회전 버튼
  - 길게 터치: 삭제

## 기술 구현 계획

### 도전 과제 및 해결
1. **컨베이어 연결 그래프**: 그리드를 방향 그래프로 변환. 각 셀의 출력 방향 → 다음 셀 입력 방향 매칭 검증.
2. **시뮬레이션 루프**: setInterval 기반. 매 틱마다 전체 그리드 순회. 역순 처리로 아이템 충돌 방지.
3. **드래그 배치**: mousedown → mousemove 이벤트로 컨베이어 연속 배치. 이전 셀→현재 셀 방향 자동 계산.
4. **실행 취소/다시 실행**: 커맨드 패턴. 배치/삭제/회전 각 동작을 스택에 push. undo 시 pop & 역작업.
5. **반응형**: 그리드 셀 크기를 화면에 맞게 동적 계산. 모바일에서는 줌/패닝 지원.
