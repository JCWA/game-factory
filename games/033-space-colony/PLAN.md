# 033 - 우주 식민지 경영 (Space Colony Manager)

## 컨셉

외계 행성에 불시착한 인류의 마지막 희망. 플레이어는 식민지 총독으로서 산소, 물, 식량, 전력이라는 4대 자원 체인을 관리하며 생존 기반을 구축한다. 개별 성격과 스킬을 가진 식민자 AI가 자율적으로 행동하며, 외계 야생동물과의 조우, 기상 이변, 무역선 도착, 반란 등 끊임없는 이벤트가 긴장감을 유지한다.

**핵심 재미**: 복합 자원 체인의 최적화 + 식민자 개개인의 드라마 + 예측 불가 이벤트 대응. 모든 시스템이 연쇄적으로 영향을 주기 때문에 작은 판단 실수가 전체 식민지를 위기에 빠뜨린다. 30분 이상 플레이하게 만드는 힘은 "이번 위기만 넘기면..."이라는 생존 루프.

## 시스템 설계

### 시스템 1: 자원 체인 (Resource Chain)

4대 핵심 자원과 4대 파생 자원으로 구성.

**핵심 자원:**
| 자원 | 기본 소모 (인당/틱) | 초기 저장량 | 최대 저장량(기본) |
|------|---------------------|------------|------------------|
| 산소 (O₂) | 0.5 | 200 | 500 |
| 물 (H₂O) | 0.3 | 150 | 400 |
| 식량 (Food) | 0.4 | 100 | 300 |
| 전력 (Power) | 0.2 (건물별) | 100 | 300 |

**파생 자원:**
| 자원 | 생산 경로 | 용도 |
|------|-----------|------|
| 광물 (Mineral) | 채광장 → 정련소 | 건물 건설, 무역 |
| 합금 (Alloy) | 광물 2 + 전력 1 → 합금 1 | 고급 건물, 무기 |
| 바이오젤 (BioGel) | 식물 연구소 → 바이오젤 | 의료, 연구 |
| 데이터 (Data) | 연구소 → 데이터 포인트 | 기술 연구 |

**자원 체인 공식:**
```
산소 생산 = Σ(산소발생기.효율 × 전력비율) - (식민자수 × 0.5) - (건물환기 × 0.1)
물 생산 = Σ(수분추출기.효율 × 지하수량) - (식민자수 × 0.3) - (농장관개 × 0.2)
식량 생산 = Σ(농장.효율 × 물공급비율 × 빛비율) - (식민자수 × 0.4)
전력 생산 = Σ(발전기.출력) - Σ(건물.전력소모)
```

**틱 시스템:** 1틱 = 2초 (실시간). 속도 조절: ×1, ×2, ×5, 일시정지.

**자원 부족 효과:**
- 산소 < 20%: 식민자 체력 -1/틱, 패닉 무드
- 물 < 15%: 농장 생산 정지, 식민자 갈증 디버프
- 식량 = 0: 식민자 체력 -2/틱, 배고픔 무드, 반란 확률 +30%
- 전력 < 10%: 모든 건물 효율 50% 감소

### 시스템 3: 식민자 AI (Colonist AI)

각 식민자는 독립적인 에이전트.

**식민자 속성:**
```javascript
colonist = {
  id: number,
  name: string,           // 랜덤 생성
  age: number,            // 20-55
  health: number,         // 0-100 (0=사망)
  hunger: number,         // 0-100 (0=만복, 100=아사)
  thirst: number,         // 0-100
  happiness: number,      // 0-100
  energy: number,         // 0-100 (0=수면 필요)
  personality: string,    // 'leader'|'worker'|'scientist'|'rebel'|'social'
  skills: {
    farming: 1-10,
    mining: 1-10,
    science: 1-10,
    combat: 1-10,
    medical: 1-10,
    construction: 1-10
  },
  assignment: string|null,  // 건물 ID 또는 null
  relationships: Map<colonistId, number>, // -100~+100
  status: string,         // 'working'|'resting'|'eating'|'socializing'|'panicking'|'rebelling'
  traits: string[],       // ['낙관적','야심가','게으름뱅이','천재','겁쟁이',...]
  sickDays: number        // 질병 남은 틱
}
```

**성격별 행동 가중치:**
| 성격 | 작업효율 | 사회활동 | 반란 가능성 | 특수 |
|------|---------|---------|------------|------|
| leader | ×1.0 | ×1.5 | 0.02 | 주변 식민자 행복도 +5 |
| worker | ×1.3 | ×0.8 | 0.05 | 건설/채광 보너스 |
| scientist | ×1.0 | ×0.6 | 0.03 | 연구 속도 ×1.5 |
| rebel | ×0.8 | ×1.2 | 0.15 | 불만 전파 |
| social | ×0.9 | ×2.0 | 0.01 | 관계 형성 속도 ×2 |

**AI 행동 루프 (매 틱):**
```
1. 욕구 체크 (hunger > 70? → 식사 이동, energy < 20? → 수면)
2. 상태 업데이트:
   - hunger += 0.3 + (작업중 ? 0.2 : 0)
   - thirst += 0.4
   - energy -= 0.2 + (작업중 ? 0.3 : 0)
   - happiness += relationships_avg × 0.01 - (hunger>50 ? 0.5 : 0) - (health<50 ? 1.0 : 0)
3. 현재 상태에 따른 행동:
   - working: 할당된 건물에서 생산 (스킬 보너스 적용)
   - resting: 거주모듈에서 energy +2/틱
   - eating: 식당에서 hunger -5/틱 (식량 소모)
   - socializing: 레크리에이션에서 happiness +1/틱, 관계 변화
   - panicking: 이동 랜덤, 작업 중단
   - rebelling: 건물 파괴 시도, 다른 rebel 모집
4. 관계 업데이트:
   - 같은 건물 근무: +0.1/틱
   - 성격 상성 보너스: leader-worker +0.2, rebel-leader -0.3
   - 관계 > 60: 친구 (서로 행복도 보너스)
   - 관계 < -40: 적 (갈등 이벤트 발생 가능)
5. 반란 체크:
   - 반란확률 = personality.rebellionBase × (1 + (100-happiness)/50) × (관계<-20인 동료수 × 0.1)
   - 반란확률 > random(): 반란 상태 진입
```

**질병 시스템:**
- 매 100틱마다 질병 체크: 확률 = 0.02 × (1 + (100-health)/100) × (의료실 없으면 ×2)
- 질병 효과: 작업효율 ×0.3, 건강 -0.5/틱
- 치료: 의료실 배정 → 30틱 후 완치 (의료스킬 높으면 단축)
- 전염: 같은 건물의 식민자에게 전파 확률 0.05/틱

### 시스템 4: 외계 야생동물 (Alien Wildlife)

5종의 야생동물이 맵에 스폰.

| 종 | 크기 | HP | 공격력 | 속도 | 행동 | 길들이기 |
|----|------|-----|-------|------|------|---------|
| 글로우버그 (Glowbug) | 소 | 5 | 1 | 3 | 무해, 야간 발광 | 쉬움 (10틱) → 야간 조명 |
| 스파이크러너 (Spikerunner) | 중 | 20 | 5 | 5 | 공격적, 단독 | 보통 (30틱) → 정찰 |
| 록쉘 (Rockshell) | 대 | 50 | 3 | 1 | 수동적, 무리 | 어려움 (60틱) → 채광 보조 |
| 스톰윙 (Stormwing) | 대 | 30 | 8 | 7 | 폭풍 시 출현, 비행 | 불가 |
| 레비아탄 (Leviathan) | 거대 | 200 | 20 | 2 | 보스급, 80틱마다 | 불가 |

**전투 공식:**
```
데미지 = 공격력 × (1 + combat_skill/20) × random(0.8, 1.2)
방어 = 방어포탑 있으면 자동 사격 (10데미지/틱, 사거리 3칸)
식민자 전투: combat_skill × 2 데미지, health로 HP 대체
```

**길들이기:**
- 바이오젤 1 소모/틱 during taming
- combat_skill 5 이상 필요 (스파이크러너), 7 이상 (록쉘)
- 성공 시 식민지 소속으로 전환, 특수 보너스 제공

**스폰 규칙:**
- 소형: 20틱마다 맵 가장자리에 1-3마리
- 중형: 40틱마다 1-2마리
- 대형: 80틱마다 1마리
- 레비아탄: 200틱마다 출현, 경고 10틱 전 알림

## AI 설계

### 식민자 AI 상태머신

```
IDLE → 욕구 최우선 체크:
  hunger > 70 → EATING (식량 있으면)
  thirst > 70 → DRINKING (물 있으면)
  energy < 20 → SLEEPING
  health < 30 → SEEKING_MEDICAL (의료실 있으면)
  
욕구 만족 시:
  assignment != null → WORKING (할당 건물로 이동)
  assignment == null → 
    happiness < 30 → SOCIALIZING (레크리에이션 찾기)
    else → IDLE (배회)

특수 전이:
  야생동물 공격 → FLEEING (가장 가까운 건물로) or FIGHTING (combat>5)
  반란 트리거 → REBELLING
  기상 위험 → SHELTERING
```

### 야생동물 AI

```
WANDERING: 랜덤 이동 (2-4칸/틱)
  식민지 감지(거리 5 이하) →
    공격적 종: APPROACHING → ATTACKING
    수동적 종: FLEEING
    
ATTACKING: 가장 가까운 식민자/건물 공격
  HP < 30% → FLEEING
  대상 파괴 → WANDERING

TAMED: 길들여진 상태
  주인 따라다니기, 명령 수행
```

### 식민자 목록: 테이블 (이름, 상태, 스킬바, 행복도 게이지, 할당)
### 무역 화면: 상품 목록 + 수량 조절 + 가격 표시

## 저장 시스템

**localStorage 키:** `spaceColony_save_{slot}` (slot: 1-3)

```javascript
saveData = {
  version: '1.0',
  slot: 1,
  timestamp: Date.now(),
  playTime: 0,              // 총 플레이 시간 (초)
  gameState: { /* 전체 gameState 직렬화 */ },
  settings: {
    soundEnabled: true,
    soundVolume: 0.5,
    autoSaveInterval: 300,   // 5분마다 자동 저장
    tutorialDone: false
  }
};
```

**자동 저장:** 300초(150틱)마다 슬롯 0(자동)에 저장.
**저장 슬롯 UI:** 3개 수동 슬롯 + 1개 자동 슬롯. 각 슬롯에 일수, 인구, 플레이시간 표시.
**불러오기:** 슬롯 선택 → 확인 → gameState 복원 → 게임 루프 재시작.

## 기술 도전 및 해결

### 성능 최적화
- **타일맵 렌더링:** 더티 플래그 패턴. 변경된 타일만 재렌더링. 전체 맵을 오프스크린 캔버스에 캐싱.
- **식민자 AI:** 10명 이상이면 틱당 절반만 업데이트 (라운드 로빈). 경로탐색은 BFS (맵이 작으므로 충분).
- **이벤트 루프:** requestAnimationFrame 기반, 틱 간격 = 2000ms / speed.

### 메모리 관리
- gameState 전체 크기 예상: ~50KB (식민자 40명, 건물 50개 기준)
- localStorage 제한 (5MB) 내 충분 (세이브 3슬롯 = ~200KB)
- 이벤트 로그: 최근 50개만 유지, 이전 기록 삭제

### 경로탐색
- BFS 기반 최단경로 (20×15 맵에서 최대 300타일 → 무시할 수준)
- 캐싱: 동일 출발-도착 경로 5틱간 캐싱
- 건물을 장애물로 처리 (통과 불가)

### 맵 생성
- 심플렉스 노이즈 2D: 바이옴 결정 (임계값으로 6개 바이옴 분류)
- 광맥: 랜덤 5-8개 위치, 암석지대 우선
- 시작 위치: 맵 중앙 근처 평원 보장 (3×3 평원 강제)

### 반응형 디자인
- 캔버스 크기: 뷰포트 비율에 맞게 조정
- 모바일: 터치 드래그로 맵 스크롤, 핀치 줌 (확대/축소)
- PC: 마우스 드래그 + 스크롤 휠 줌
- 정보 패널: 모바일에서는 하단 드로어로 전환
