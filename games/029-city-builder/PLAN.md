# 029 - 절차적 도시 빌더

## 컨셉
도로를 배치하면 건물이 자동으로 생성되는 도시 건설 시뮬레이션. SimCity의 핵심 시스템을 단일 HTML에 구현. 인구 성장, 행복도, 경제, 교통, 재난 이벤트까지 포함된 복합 시뮬레이션. 도로와 구역(주거/상업/공업)을 지정하면 AI가 건물을 자동으로 성장시키고, 시민들이 출퇴근하며, 경제가 돌아간다.

핵심 재미: "도시가 살아있다"는 느낌. 도로 하나를 놓으면 건물이 자라고 사람들이 모이는 유기적 성장의 쾌감. 예산 관리의 긴장감.

## 핵심 시스템

- **구역 (Zone) 시스템**:

### 시스템 2: 건물 자동 성장 (절차적 생성 AI)
도로 인접 구역 타일에 건물이 자동 생성/성장:

```
성장 조건 계산 (매 시뮬 틱):
  for each 구역 타일:
    if 타일에 건물 없음:
      수요 = calculateDemand(구역타입)
      if 수요 > 0 && random() < 수요 * 0.1:
        건물 생성 (레벨 1)
    
    if 타일에 건물 있음 && 건물.레벨 < 3:
      만족도 = calculateSatisfaction(타일)
      if 만족도 > 0.7 && random() < 만족도 * 0.05:
        건물 레벨업 (더 크고 높은 건물)
      if 만족도 < 0.3 && random() < 0.02:
        건물 퇴화 (레벨다운 또는 폐허)

수요 계산:
  주거 수요 = (일자리 수 - 인구) / 일자리 수  (0~1)
  상업 수요 = (인구 - 상업건물 수 * 20) / 인구  (0~1)  
  공업 수요 = (상업건물 수 - 공업건물 수 * 2) / 상업건물 수  (0~1)

만족도 계산 (0~1):
  기본 = 0.5
  + 전력 공급 ? 0.15 : -0.3
  + 물 공급 ? 0.1 : -0.2
  + 교통 혼잡도 < 0.5 ? 0.1 : -0.1
  + 범죄율 < 0.3 ? 0.05 : -0.1
  + 공원 근처 ? 0.05 : 0
  + 공업 근처 (주거만) ? -0.15 : 0
```

- **건물 레벨 & 비주얼**:
  | 구역 | 레벨 1 | 레벨 2 | 레벨 3 |
  |------|--------|--------|--------|
  | 주거 | 작은 집 (1층) | 다세대 (2층) | 아파트 (3층) |
  | 상업 | 작은 상점 | 마트 | 백화점 |
  | 공업 | 작은 공장 | 중형 공장 | 대형 공장 |

  레벨에 따라 인구/일자리 수용량 증가:
  - 주거: L1=2명, L2=8명, L3=20명
  - 상업: L1=3일자리, L2=10일자리, L3=25일자리
  - 공업: L1=5일자리, L2=15일자리, L3=30일자리

### 시스템 3: 경제 & 예산
- **수입**:
  - 주거세: 인구 * 세율(0~20%) * 소득수준
  - 상업세: 상업건물 수 * 세율(0~20%) * 50
  - 공업세: 공업건물 수 * 세율(0~20%) * 80
  - 소득수준: 교육도에 비례 (0.5~2.0 배율)

- **지출**:
  - 특수 건물 유지비 (월별)
  - 도로 유지비: 도로 타일 수 * 0.5/월
  - 재난 복구 비용

- **세율 효과**:
  - 세율 높음 → 수입 증가, 행복도 감소, 이탈 증가
  - 세율 낮음 → 수입 감소, 행복도 증가, 유입 증가
  - 적정 세율: 8~12%

- **재정 파탄**: 예산 < -5000이면 게임 오버 경고. < -10000이면 강제 종료.

### 시스템 4: 인구 시뮬레이션
- **인구 변동**:
  ```
  월별 인구 변동 = 유입 - 이탈 + 자연증가
  유입 = 주거 빈자리 * 행복도 * 0.1
  이탈 = 인구 * max(0, (1 - 행복도)) * 0.05
  자연증가 = 인구 * 0.01
  ```

- **행복도 (0~1)**:
  ```
  행복도 = 평균(
    세금 만족도: 1 - 세율/25,
    교통 만족도: 1 - 평균 통근시간/60,
    안전 만족도: 1 - 범죄율,
    건강 만족도: 병원 커버리지,
    교육 만족도: 학교 커버리지,
    환경 만족도: 공원 커버리지 - 공해,
    서비스 만족도: (전력 + 물 + 쓰레기) / 3
  )
  ```

### 시스템 5: 재난 이벤트
- **랜덤 재난** (확률/월):
  | 재난 | 확률 | 효과 | 대응 |
  |------|------|------|------|
  | 화재 | 3% | 건물 1~3개 파괴 | 소방서 있으면 피해 -70% |
  | 지진 | 1% | 넓은 범위 건물 피해 | 피해 건물 자동 복구 (비용 발생) |
  | 홍수 | 2% | 저지대 건물 피해 | 물가 근처 건물 위험 |
  | 범죄파동 | 5% | 상업 수익 -30%, 1개월 | 경찰서로 예방 |
  | 경기침체 | 3% | 세수 -50%, 3개월 | 세율 조정으로 대응 |
  | 전염병 | 1% | 인구 -5%, 행복도 -20% | 병원으로 피해 경감 |

- **재난 시각화**: 화재=불꽃 파티클, 지진=화면 흔들림, 홍수=파란 타일 확산

## 게임 규칙
- 시청 건설이 첫 번째 목표 (시청 없으면 세금 징수 불가)
- 구역은 도로에 인접한 타일에만 설정 가능
- 건물은 전력과 물이 공급되어야 성장
- 도로 연결이 끊기면 해당 구역 만족도 급감
- 시간 속도 조절: 일시정지, x1, x2, x5
- 1게임월 = 실시간 10초 (x1 기준)
- 인구 목표 달성 시 마일스톤 보상 (예산 보너스)
  - 100명: +1000
  - 500명: +3000
  - 1000명: +5000
  - 5000명: +10000
- 초기 예산: 5000
- 도로/구역 배치는 일시정지 중에도 가능
- 특수 건물은 도로에 인접해야 건설 가능
- 같은 타입의 특수 건물 반경이 겹치면 효과 중첩 안 됨
- 불도저로 건물/도로/구역 삭제 가능 (비용 = 건설비의 30%)
- 맵 경계에는 건설 불가 (1타일 여백)

## AI 설계

### 건물 성장 AI (상태 기반)
각 구역 타일의 건물은 독립적인 상태 머신:
```
상태: 빈터 → 건설중(3틱) → 레벨1 → 성장중(5틱) → 레벨2 → 성장중(8틱) → 레벨3
         ↘ 퇴화조건 충족 시 ↙
         폐허 → 빈터 (10틱 후)

전이 조건:
  성장: 만족도 > 0.7, 수요 > 0
  퇴화: 만족도 < 0.3 (연속 5틱)
  복구: 폐허 10틱 후 자동 철거
```

### 교통 시뮬레이션 AI
```
매 시뮬 틱:
  for each 주거 건물:
    가장 가까운 상업/공업 건물까지 도로 경로 (BFS)
    경로 길이 = 통근 시간
    경로 상 도로 타일의 교통량 +1
  
  교통 혼잡도 = 도로타일.교통량 / 도로타일.용량
  혼잡도 > 1 이면 빨간색으로 표시
```

### 어드바이저 AI
상황에 따라 조언 제공:
```
- 주거 수요 높음 → "주거 구역이 부족합니다. 더 지정하세요."
- 전력 부족 → "전력이 부족합니다! 발전소를 건설하세요."
- 범죄율 높음 → "범죄율이 증가하고 있습니다. 경찰서가 필요합니다."
- 재정 적자 → "예산이 부족합니다. 세율을 올리거나 지출을 줄이세요."
- 교통 혼잡 → "교통이 혼잡합니다. 대로를 건설하세요."
```

## 조작법
- **PC**:
  - 좌측 도구 선택 → 맵 클릭/드래그로 배치
  - R: 도로 모드
  - Z: 구역 모드 (1=주거, 2=상업, 3=공업)
  - B: 건물 모드
  - X: 불도저 모드
  - Space: 일시정지/재개
  - +/-: 속도 조절
  - 마우스 휠: 줌
  - 마우스 드래그 (우클릭): 패닝
  - I: 통계/그래프 패널 토글
- **모바일**:
  - 하단 도구바에서 선택 → 탭으로 배치
  - 드래그: 연속 배치 (도로/구역)
  - 핀치: 줌
  - 두 손가락 드래그: 패닝
  - 속도/일시정지 버튼

- **애니메이션**:
  - 건물 생성: 아래에서 위로 솟아오름 (300ms)
  - 건물 파괴: 붕괴 + 먼지 파티클
  - 재난: 화재(불꽃 파티클), 지진(화면 흔들림), 홍수(파란 물결)
  - 교통: 혼잡한 도로 빨간 펄스

## 레벨/콘텐츠 설계
- 자유 모드 (샌드박스) — 레벨 없음
- **시나리오 모드 3개**:
  1. "첫 번째 도시" — 인구 500 달성 (튜토리얼)
  2. "경제 도시" — 예산 20000 달성
  3. "행복한 도시" — 행복도 90% 이상 유지 (12개월)

## 기술 구현 계획

### 도전 과제 및 해결
1. **도로 자동 타일링**: 비트마스크 방식. 상하좌우 4방향 인접 도로 유무를 4비트로 인코딩. 16가지 경우의 수에 대응하는 렌더링.
2. **서비스 범위 계산**: BFS로 특수 건물에서 반경 내 타일에 서비스 플래그 설정. 건물 건설/삭제 시 재계산.
3. **교통 시뮬**: 전체 BFS는 무겁므로 샘플링. 주거 건물의 10%만 매 틱 경로 계산. 결과를 외삽.
4. **성능**: 1200타일 그리드. dirty rect로 변경 부분만 다시 그리기. 시뮬 계산은 Web Worker 없이도 충분 (단순 연산).
5. **반응형**: 타일 크기를 화면에 맞게 조절. 모바일에서는 줌 기본값을 축소.
